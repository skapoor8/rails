    Introduction

    Welcome to the Dispatch API! You can use our API to access Dispatch API endpoints, which can get information on various incidents, users, and departments in our database.

    We have language bindings in Shell, Ruby, and Python! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    This example API documentation page was created with Slate. Feel free to use it as a base for your own API’s documentation. http://derek.broox.com/using-curl-to-test-restful-rails-web-services/

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "api_endpoint_here"
  -H "Authorization: developerKey"

    Make sure to replace developerKey with your API key.

    Dispatch uses API keys to allow access to the API. You can register a new Dispatch API key at our developer portal.

    Dispatch expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: developerKey

    You must replace developerKey with your personal API key.

    URL Parameters

    Parameter Required? Description
    ID Yes The ID of the incident to retrieve
    location No String value describing the incident’s location
    report_date No String timestamp value
    start_date No String timestamp value
    checkout_date No String timestamp value
    resolve_date No String timestamp value
    resolve_date No String timestamp value

    severity No Integer value from 1 - 10 representing minimum to maximum seriousness of incident
    bump_count No Integer value representing number of users who have endorsed/bumped this reported incident.
    user_id No Integer value corresponding to primary key of user who reported incident.

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`’s frontmatter. Files are included in the order listed.

    The Dispatch API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The resource requested is hidden for administrators only
    404 Not Found – The specified resource could not be found
    405 Method Not Allowed – You tried to access a resource with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The resource requested has been removed from our servers
    429 Too Many Requests – You’re requesting too many resources! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
